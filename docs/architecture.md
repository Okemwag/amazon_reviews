# Project Structure

The repository is organized as a small data application, not a collection of
independent scripts.

```text
config/
  pipeline.toml                 Runtime settings for Docker, HDFS, Hive, Spark.

src/amazon_reviews_pipeline/
  pipeline.py                   End-to-end orchestrator.
  jobs/
    transform_reviews.py        Spark cleaning and feature engineering.
    quality_checks.py           Spark validation checks and quality report.
    train_sentiment_model.py    Spark MLlib model training.
  sql/
    create_tables.sql           Hive external table DDL.
    analysis.sql                Hive analytical queries.

data/
  raw/                          Downloaded Amazon Reviews JSONL files.
  manifests/                    Per-ingestion manifests generated by pipeline.
  processed/                    Local quality and model metric outputs.
```

Only `src/amazon_reviews_pipeline/common/pipeline.py` should orchestrate pipeline stages. New
behavior belongs in package modules under `src/amazon_reviews_pipeline`, not in ad hoc
top-level shell scripts.
